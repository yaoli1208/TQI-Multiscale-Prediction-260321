## 一、铁路轨道检测与TQI相关项目

### 1. UFATD - 超快速自适应轨道检测网络

| 项目信息 | 详情 |
|---------|------|
| **论文标题** | Ultra-Fast Adaptive Track Detection Network |
| **源码地址** | https://github.com/idnihai/UFATD |
| **发布时间** | 2024年5月 |
| **机构** | 中南大学交通运输工程学院 |
| **联系邮箱** | idnihai@csu.edu.cn |

**核心内容：**
- 轻量级自适应铁路轨道检测模型
- 两个专门分支：水平坐标定位器(HCL) + 视角识别器(PI)
- 轻量版在SRail数据集上F1得分98.68%，检测速度473 FPS
- 包含自建数据集SRail（5,903张图像，含合成+真实图像）

**可用资源：**
- ✅ 完整源代码
- ✅ SRail数据集
- ✅ RailDB和DL-RAIL数据集链接
- ✅ 预训练模型

### 2. Rail-Detection - Rail-DB数据集与Rail-Net检测方法

| 项目信息 | 详情 |
|---------|------|
| **论文会议** | ACM MM 2022 |
| **源码地址** | https://github.com/Sampson-Lee/Rail-Detection |
| **发布时间** | 2022年3月 |

**核心内容：**

- 图像涵盖不同光照、道路结构、视角条件
- Rail-Net：基于行的轻量级轨道检测方法
- 将轨道检测问题转化为基于行的选择问题

**可用资源：**
- ✅ 训练脚本和代码
- ✅ Rail-DB数据集（需填表申请）
- ✅ 环境配置文件

**使用方式：**
```bash
git clone git@github.com:Sampson-Lee/Rail-Detection.git
conda env create -f environment.yaml
bash launch_training.sh
```

### 3. RT-FE - 铁轨特征提取与语义分割

| 项目信息 | 详情 |
|---------|------|
| **源码地址** | https://github.com/AnanthaPadmanaban-KrishnaKumar/RT-FE |
| **发布时间** | 2023年11月 |

**核心内容：**
- 利用深度学习从铁路轨道精确提取特征
- 基于RailSem19数据集（8,500+序列）
- 使用UNET+VGG19架构进行语义分割
- 应用：异常检测、自主铁路安全系统

**技术流程：**
视频输入 → 帧提取 → 图像预处理 → UNET分割 → 后处理 → 掩膜生成

**可用资源：**
- ✅ 完整源代码
- ✅ RailSem19数据集链接

### 5. Rail3D - 铁路语义分割点云数据集

| 项目信息 | 详情 |
|---------|------|
| **论文** | Multi-Context Point Cloud Dataset and Machine Learning for Railway Semantic Segmentation (Infrastructures, 2024) |
| **源码地址** | https://github.com/akharroubi/Rail3D |
| **发布时间** | 2023年11月 |
| **DOI** | https://doi.org/10.3390/infrastructures9040071 |

**核心内容：**
- 首个多上下文铁路点云数据集
- 覆盖4个国家：匈牙利、法国、比利时、荷兰
- 超过2.88亿个标注点
- 9个通用类别：地面、植被、轨道、电线杆、电线、信号、围栏、设施、建筑物
- 评估模型：KPConv (mIoU 86%)、LightGBM (mIoU 71%)

**数据来源：**
| 国家 | 数据提供方 | 数据量 | 格式 |
|------|-----------|--------|------|
| 法国 | SNCF Reseau | ~2公里 | LAZ |
| 比利时 | Infrabel | 多点云 | LAS |
| 荷兰 | spoorinbeeld.nl | 双区域 | PLY |
| 匈牙利 | 学术研究 | - | - |

**可用资源：**
- ✅ 源代码
- ✅ 法国、荷兰、匈牙利数据公开
- ✅ 比利时数据可申请

## 六、使用建议

### 针对用户场景(14天周期、200m区段)的建议

1. **数据预处理参考**
   - 使用 `limingwu8/Predictive-Maintenance` 中的时间序列采样方法
   - 参考电池退化预测中的多步预测实现

2. **时空建模参考**
   - 图像领域：参考UFATD的自适应检测方法
   - 点云领域：参考Rail3D的KPConv多尺度特征提取

3. **预测模型基线**
   - 短期预测：直接使用LSTM/GRU（pmx_data仓库提供模板）
   - 长期预测：参考3D CNN的时空特征融合思想

4. **知识图谱构建参考**
   - 铁路故障知识抽取可参考Rail3D的标注体系
   - 维修决策逻辑可参考预测性维护Agent架构

## 七、注意事项

1. **数据许可**
   - Rail3D的法国数据采用ODbL许可证
   - 部分数据集需向作者申请使用
   - 商业使用前请确认许可证条款

2. **代码兼容性**
   - 早期项目(2017-2019)使用TensorFlow 1.x
   - 近期项目(2023+)多使用PyTorch
   - 注意Python版本和依赖包版本匹配

3. **论文复现**
   - 部分论文仅提供数据集，不提供训练代码
   - 论文中的超参数可能需要调优才能复现结果
   - 建议先运行提供的预训练模型验证环境

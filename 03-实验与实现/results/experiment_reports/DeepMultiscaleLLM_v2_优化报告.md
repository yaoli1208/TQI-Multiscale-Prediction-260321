## 2. v1 的问题诊断

### 2.1 失败原因

| 问题 | 说明 | 影响 |
|------|------|------|
| **STL分解简单** | 使用简单移动平均，趋势不平滑 | 趋势/季节分离不清 |
| **无多尺度特征** | 只有原始序列 | 丢失多频率信息 |
| **固定融合权重** | 0.4/0.4/0.2 手工设定 | 不适应数据分布 |
| **无维修感知** | 不区分维修前/后 | 对新分布不适应 |
| **无稳定期特化** | 全年样本同等对待 | 季节噪声干扰 |

### 2.2 v1 架构缺陷
```
v1 架构:
输入 → 简单STL → 三分支独立 → 固定权重融合 → 输出
     ↓              ↓              ↓
   信息损失      无交互学习      权重不合理
```

## 3. v2 优化策略

### 3.1 五大改进

#### 改进1: STL分解优化
```python
# v1: 简单移动平均
trend = moving_average(data, window=26)

# v2: Savitzky-Golay滤波
trend = savgol_filter(data, window=51, polyorder=3)
```
**效果**: 更好地保持趋势形状，减少过平滑

#### 改进2: 多尺度特征引入 (借鉴TimeMixer)
```python
# v2新增:
features += [
    MA7,    # 7天移动平均
    MA14,   # 14天移动平均
    MA28,   # 28天移动平均
]
```
**效果**: 捕捉多频率模式，提升趋势分支性能

#### 改进3: 维修感知 + 稳定期加权
```python
# 样本权重设计
sample_weights = np.ones(n)
sample_weights[维修后样本] = 2.0      # 重点关注新分布
sample_weights[7-9月样本] *= 1.5     # 稳定期更可靠
```
**效果**: 让模型适应维修后的TQI分布

#### 改进4: 自适应融合权重
```python
# v1: 固定权重
weights = [0.4, 0.4, 0.2]

# v2: 基于训练误差自适应
weights = [0.515, 0.306, 0.179]  # 学习得到
```
**效果**: 趋势分支权重最高(0.515)，符合业务直觉

#### 改进5: 时间特征增强
```python
# v2新增特征:
features += [
    距2021年,           # 维修时间标记
    sin(2π*月份/12),    # 月份编码
    cos(2π*月份/12),
]
```
**效果**: 显式引入季节性和维修信息

## 4. v2 架构详解

### 4.1 完整架构图

```
输入: TQI序列 (52步)
    ↓
┌────────────────────────────────────────────────────────────────┐
│ Layer 1: 改进STL分解                                            │
│  ├── 趋势: Savitzky-Golay滤波 (window=51, poly=3)               │
│  ├── 季节: 26天周期平均                                          │
│  └── 残差: 原始 - 趋势 - 季节                                    │
└──────────────────┬─────────────────────────────────────────────┘
                   ↓
┌────────────────────────────────────────────────────────────────┐
│ Layer 2: 特征工程                                               │
│  ├── 各分量最后14步                                              │
│  ├── 多尺度MA: MA7/MA14/MA28                                    │
│  ├── 维修标记: 距2021年                                          │
│  └── 季节编码: sin/cos(月份)                                     │
└──────────────────┬─────────────────────────────────────────────┘
                   ↓
┌────────────────────────────────────────────────────────────────┐
│ Layer 3: 三分支MLP (加权训练)                                    │
│  ├── 趋势分支: MLP(64,32) + 20维特征 → 自适应权重0.515          │
│  ├── 季节分支: MLP(32,16) + 16维特征 → 自适应权重0.306          │
│  └── 残差分支: MLP(32,16) + 16维特征 → 自适应权重0.179          │
└──────────────────┬─────────────────────────────────────────────┘
                   ↓
┌────────────────────────────────────────────────────────────────┐
│ Layer 4: 自适应融合                                             │
│  └── 融合 = 0.515×趋势 + 0.306×季节 + 0.179×残差                 │
└──────────────────┬─────────────────────────────────────────────┘
                   ↓
输出: 26步TQI预测
```

### 4.2 分支详情

| 分支 | 输入维度 | 网络结构 | 学习权重 | 作用 |
|------|----------|----------|----------|------|
| 趋势 | 20 | (64,32) | 0.515 | 捕捉年际劣化趋势 |
| 季节 | 16 | (32,16) | 0.306 | 捕捉周期性波动 |
| 残差 | 16 | (32,16) | 0.179 | 捕捉异常/维修效应 |

**关键发现**: 趋势分支权重最高(0.515)，验证了TQI预测以趋势为主、季节为辅的业务特性。

- TimeMixer精度略高(高18%)
- DeepMultiscaleLLM v2可解释性更强
- 两者可互补使用

## 8. 业务价值

### 8.1 可解释性输出

v2可输出各分量预测，支持业务分析:
```python
预测TQI = 0.515×趋势 + 0.306×季节 + 0.179×残差

示例 (2024-07):
├── 趋势预测: 2.80 (年际劣化基线)
├── 季节预测: -0.60 (夏季低谷)
├── 残差预测: -0.20 (维修效应)
└── 总预测: 2.00 ✅
```

### 8.2 维修决策支持

- **趋势分支**: 判断是否需要维修 (年劣化速率)
- **季节分支**: 选择最佳维修时机 (避开夏季)
- **残差分支**: 评估维修效果 (残差突降)

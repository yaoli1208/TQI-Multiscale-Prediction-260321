## 1. 业务背景与关键假设

### 1.1 TQI劣化的自然规律

```
TQI年度变化规律:
┌─────────────────────────────────────────────────────────────┐
│ 正常情况下: TQI逐年劣化 (轨道自然老化)                        │
│   年劣化速率 ≈ 0.1-0.3 (视交通量、环境而定)                   │
│                                                             │
│ 异常情况: TQI年度下降                                        │
│   → 99%概率: 人工维修干预                                    │
│   → 1%概率: 检测误差或特殊环境因素                            │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 季节性稳定期 (关键业务洞察)

**7-9月稳定期特征**:
| 特征 | 说明 | 业务意义 |
|------|------|----------|
| 气温稳定 | 夏季高温稳定 | 轨道热胀冷缩处于平衡 |
| 降水稳定 | 雨季但相对稳定 | 路基含水率稳定 |
| 维修集中 | 夏季是维修黄金期 | 大量捣固、换轨作业 |
| TQI低谷 | 年度TQI最低区间 | 维修后最佳状态 |

**为什么7-9月数据最能反映真实趋势**:
- 排除冬季冻胀干扰
- 排除春季融沉干扰
- 维修效果已稳定显现
- 检测条件一致性好

### 1.3 维修事件的影响

```
维修前后的TQI变化模式:

维修前: TQI持续上升 (劣化趋势)
   ↓
维修点: 捣固/换轨/打磨作业
   ↓
维修后: TQI突降，然后缓慢回升
   │
   ├── 短期(1-2月): 快速回升期 (ΔTQI +0.3~0.5)
   │
   └── 长期(3-12月): 缓慢劣化期 (ΔTQI +0.1~0.2/月)
```

```python
def detect_maintenance_events(df, threshold=0.3):
    """
    检测可能的维修事件
    规则:
    1. 连续两次检测TQI下降超过threshold
    2. 或年度7-9月TQI同比下降超过threshold
    """
    events = []
    
    # 规则1: 短期突降
    df['TQI变化'] = df['TQI值'].diff()
    sudden_drops = df[df['TQI变化'] < -threshold]
    
    # 规则2: 年度优化 (反常)
    yearly_stable = df[df['月份'].isin([7,8,9])].groupby('年份')['TQI值'].mean()
    yearly_change = yearly_stable.diff()
    maintenance_years = yearly_change[yearly_change < -0.2].index.tolist()
    
    return sudden_drops, maintenance_years
```

## 3. 业务驱动的模型优化策略

### 3.1 分段建模策略

**核心思想**: 维修是趋势的分水岭，应分段训练

```
全局趋势 (2012-2026)
    ├── 阶段1 [2012-2020]: 自然劣化期
    │       └── 模型A: 劣化趋势预测
    │
    └── 阶段2 [2021-2026]: 维修优化期
            └── 模型B: 维护状态预测

维修点 (2021年初)
    └── 触发模型切换
```

**实现方案**:

```python
class SegmentedTQI Predictor:
    def __init__(self):
        self.model_natural = None  # 劣化期模型
        self.model_maintained = None  # 维护期模型
        self.maintenance_threshold = 0.3
        
    def fit(self, df):
        # 检测维修点
        maintenance_points = self._detect_maintenance(df)
        
        if maintenance_points:
            split_idx = maintenance_points[0]
            # 分段训练
            self.model_natural.fit(df[:split_idx])
            self.model_maintained.fit(df[split_idx:])
        else:
            self.model_natural.fit(df)
    
    def predict(self, current_tqi, days_since_maintenance):
        """
        根据距维修时间选择模型
        """
        if days_since_maintenance < 365*2:
            # 维修后2年内: 使用维护模型
            return self.model_maintained.predict(current_tqi)
        else:
            # 维修后2年+: 使用自然劣化模型
            return self.model_natural.predict(current_tqi)
```

**预期效果**:
- 解决分布偏移问题 (训练集包含两个不同分布)
- 提高维修后短期预测精度
- 更好捕捉长期劣化趋势

### 6.1 短期优化 (1周内)

3. **分段验证**
   - 分别验证维修前/后模型的精度

### 6.2 中期优化 (2-4周)

1. **实现分段建模**

3. **情景预测功能**

## 8. 关键结论

### 8.1 核心洞察

1. **7-9月是黄金评估期** - 应优先用稳定期数据评估年度趋势
2. **维修是趋势分水岭** - 必须分段建模，不能用单一模型
3. **年度TQI下降=维修信号** - 业务规则可辅助检测维修事件
4. **预测要考虑情景** - 维修/不维修两种情景都要提供

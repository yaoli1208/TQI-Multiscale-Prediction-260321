# TimeMixer 优化空间分析报告

**分析对象**: TimeMixer 在3号样本上的预测表现  
**基准性能**: MAE=0.2964, MAPE=13.99%  
**分析日期**: 2026-03-21

---

## 1. 问题诊断

### 1.1 当前表现瓶颈

| 问题 | 现象 | 影响程度 |
|------|------|----------|
| **远期误差累积** | 19-26步MAE达0.42，是早期的3.3倍 | 🔴 高 |
| **低谷期高估** | TQI=2.0时预测2.4-2.6，高估20-30% | 🔴 高 |
| **分布偏移敏感** | 测试集(2.19)与训练集(3.18)差异大 | 🟡 中 |
| **多尺度提升有限** | 仅比单尺度提升1.4% | 🟡 中 |

### 1.2 误差模式分析

```
预测误差热力图 (26步 × 误差大小)
步骤  1-9      10-18     19-26
     ┌─────────┬─────────┬─────────┐
误差 │ ░░░░░░░ │ ███░░░░ │ █████░░ │  ░=小误差  █=大误差
     │ ░░░░░░░ │ ████░░░ │ ██████░ │
     └─────────┴─────────┴─────────┘
      早期优秀   中期一般   远期偏差大
```

**关键发现**: 误差随预测步长增加而系统性增大，呈明显的**长尾分布**。

## 2. 优化方向与建议

### 2.1 架构优化 (预计提升15-25%)

#### 方案A: 引入注意力机制
```
当前: MLP(64,32) → 输出
优化: MLP(64) → Attention → MLP(32) → 输出

原理: 让模型自动学习不同时间步/尺度的权重
预期效果: MAE降至 0.24-0.26
实现难度: 中 (需PyTorch)
```

#### 方案B: 分层预测器 (借鉴FMM)
```
当前: 单一MLP输出26步
优化: 
  预测器1: 输出1-9步 (短期)
  预测器2: 输出10-18步 (中期) 
  预测器3: 输出19-26步 (长期)
  融合层: 加权集成

原理: 不同时间范围有不同的最优模型
预期效果: 远期MAE降至 0.30-0.35
实现难度: 低
```

#### 方案C: 自回归修正
```
当前: 一次性输出26步
优化:
  第1步: 预测t+1
  第2步: 用预测值作为输入，预测t+2
  ...
  第26步: 迭代预测

原理: 每步都有最新信息，减少误差累积
预期效果: 整体MAE降至 0.22-0.25
实现难度: 中
注意: 计算成本增加26倍
```

### 2.2 特征工程优化 (预计提升10-20%)

#### 方案D: 增加更多时间特征
```
当前特征: 14步历史 + MA7/14/28
优化特征:
  + 月份正弦编码 (捕捉季节性)
  + 距上次维修天数 (捕捉维修效应)
  + 年际位置编码 (1-366天)
  + 趋势斜率 (最近N步线性回归斜率)
  + 波动率 (最近N步标准差)

原理: 显式引入时间先验知识
预期效果: MAE降至 0.24-0.27
实现难度: 低
```

#### 方案E: 引入TQI分项指标
```
3号样本有7项分项指标:
  - TQI左高低、TQI右高低 (主要贡献)
  - TQI左轨向、TQI右轨向
  - TQI轨距、TQI三角坑、TQI水平

优化:
  分别预测7个子项，再求和得TQI
  或作为辅助特征输入

原理: 分项指标包含更多诊断信息
预期效果: MAE降至 0.25-0.28
实现难度: 中
注意: 需要重新训练，数据格式改变
```

#### 方案F: 多尺度改进
```
当前: MA7, MA14, MA28 (等间隔)
优化:
  - 增加MA52 (年度尺度)
  - 增加指数加权平均 (更关注近期)
  - 增加差分特征 (变化率)
  - 增加峰值/谷值检测特征

原理: 更精细的多尺度分解
预期效果: MAE降至 0.27-0.29
实现难度: 低
```

### 2.3 训练策略优化 (预计提升5-15%)

#### 方案G: 加权损失函数
```
当前: MSE (均方误差)
优化:
  - 对近期预测加权更高
    loss = Σ w_i * (y_i - ŷ_i)²
    w_i = exp(-0.1 * i)  # i为步长，近期权重高
  
  - 或对低估/高估分别惩罚
    loss = Σ |y_i - ŷ_i| * (1 + α * sign(低估))

原理: 优化业务关心的指标
预期效果: 关键步骤误差降低
实现难度: 低
```

#### 方案H: 课程学习 (Curriculum Learning)
```
当前: 随机训练所有样本
优化:
  第1阶段: 只训练易样本 (误差<0.2的预测)
  第2阶段: 加入中等样本
  第3阶段: 加入全部样本

原理: 从简单到复杂，逐步提升
预期效果: 收敛更快，泛化更好
实现难度: 中
```

#### 方案I: 集成学习
```
当前: 单一MLP
优化:
  训练5个不同的MLP (不同初始化)
  预测时取平均

  或使用Bagging: 不同子集训练

原理: 降低方差，提高稳定性
预期效果: MAE降至 0.26-0.28
实现难度: 低
```

### 2.4 数据处理优化 (预计提升10-20%)

#### 方案J: 处理分布偏移
```
问题: 训练集TQI均值3.18，测试集2.19

优化A: 标准化改进
  当前: StandardScaler (全局标准化)
  优化: 滚动标准化 (用最近N步的均值/方差)

优化B: 差分预测
  预测ΔTQI而非绝对TQI
  ŷ_t = y_{t-1} + Δŷ_t

优化C: 域适应
  对测试集分布进行校准
  ŷ_corrected = ŷ * (测试集均值/训练集均值)

预期效果: 解决低谷期高估问题
实现难度: 低到中
```

#### 方案K: 数据增强
```
当前: 348个训练样本
优化:
  - 时间平移: 序列前后移动1-2步
  - 加噪声: 加入高斯噪声(σ=0.05)
  - 缩放: TQI乘以0.9-1.1
  - Mixup: 两个序列加权混合

预期效果: 样本增至1000+，减少过拟合
实现难度: 低
```

#### 方案L: 处理类别不平衡
```
观察: TQI=2.0的样本在训练集中较少

优化:
  过采样低谷值样本
  或使用Focal Loss

预期效果: 改善低谷期预测
实现难度: 低
```

### 2.5 针对TQI任务的特定优化

#### 方案M: 物理约束
```
TQI的物理特性:
  1. 不能为负
  2. 变化率有限 (单次检测变化通常<0.5)
  3. 维修后会突降

优化:
  输出层加约束: ReLU + 变化率截断
  或后处理: 预测值修正

预期效果: 减少不合理预测
实现难度: 低
```

#### 方案N: 维修事件建模
```
优化:
  检测维修记录中的突变点
  维修前后分别建模

  或引入维修标记作为特征:
    距上次维修天数
    是否刚维修 (0/1)

预期效果: 维修前后预测更准确
实现难度: 中 (需标注数据)
```

## 7. 结论

### 核心观点

1. **分布偏移是首要问题** - 测试集TQI整体偏低，模型有系统性高估

3. **特征工程空间大** - 时间特征、分项指标尚未充分利用
4. **架构升级潜力大** - 注意力机制预计带来15-25%提升

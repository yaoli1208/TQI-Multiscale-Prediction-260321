## 2. 融合架构设计

### 2.1 三层融合架构

```
┌─────────────────────────────────────────────────────────────────────┐
│                    多方法融合架构 (Multi-Method Fusion)              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  Level 1: 基模型层 (Base Models)                                    │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │ Model A: TimeMixer风格                                       │   │
│  │ ├── 输入: 52步历史 + MA7/MA14/MA28                          │   │
│  │ ├── 网络: MLP(64,32)                                        │   │
│  │ └── 特点: 纯数据驱动，捕捉多尺度模式                        │   │
│  ├─────────────────────────────────────────────────────────────┤   │
│  │ Model B: DeepMultiscaleLLM v2 (STL分解)                      │   │
│  │ ├── 分解: Savitzky-Golay趋势 + 周期季节 + 残差              │   │
│  │ ├── 增强: 距2021年 + 月份编码                               │   │
│  │ └── 特点: 显式建模趋势，维修感知                            │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                              ↓                                      │
│  Level 2: 融合层 (Fusion Layer)                                      │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │ 策略1: 简单平均 (Simple Average)                             │   │
│  │   pred = 0.5 × A + 0.5 × B                                  │   │
│  ├─────────────────────────────────────────────────────────────┤   │
│  │ 策略2: 误差加权 (Error-Weighted)                             │   │
│  │   w_a = 1/MAE_a, w_b = 1/MAE_b                              │   │
│  │   pred = w_a×A + w_b×B / (w_a+w_b)                          │   │
│  ├─────────────────────────────────────────────────────────────┤   │
│  │ 策略3: 分步融合 (Stepwise Fusion)                            │   │
│  │   每步独立选择最佳模型 (基于验证集)                         │   │
│  ├─────────────────────────────────────────────────────────────┤   │
│  │ 策略4: 规则融合 (Rule-Based)                                 │   │
│  │   1-9月: B权重高(维修感知)                                  │   │
│  │   10-26步: A权重高(趋势外推)                                │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                              ↓                                      │
│  Level 3: 后处理层 (Post-Processing)                                 │
│  ├── 物理约束: TQI>0, 变化率<0.3                                   │
│  └── 平滑处理: 移动平均去噪                                        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 3. 关键发现与分析

### 3.1 Model B 为何表现最佳？

**核心优势**:

| 因素 | Model A | Model B | 影响 |
|------|---------|---------|------|
| **STL分解** | ❌ 无 | ✅ Savitzky-Golay | 趋势分离更清晰 |
| **维修感知** | ❌ 无 | ✅ 距2021年特征 | 适应分布变化 |
| **季节编码** | ❌ 无 | ✅ sin/cos月份 | 捕捉周期性 |
| **样本加权** | ❌ 均匀 | ✅ 维修后×2 | 关注新分布 |
| **稳定期加权** | ❌ 无 | ✅ 7-9月×1.5 | 高可信度样本 |

**结论**: 显式建模趋势 + 维修感知 + 季节编码 = 最佳性能

## 4. 最佳模型详解 (Model B)

### 4.1 架构细节

```python
# 输入: 52步历史TQI
Input (52,) 
    ↓
# STL分解
┌─────────────────────────────────────┐
│ trend = savgol_filter(input, 51, 3) │  # 平滑趋势
│ seasonal = cycle_average(detrended) │  # 26天周期
│ residual = input - trend - seasonal │  # 随机成分
└─────────────────────────────────────┘
    ↓
# 特征构建
┌─────────────────────────────────────┐
│ features = [                        │
│   trend[-14:],                      │  # 趋势最后14步
│   MA7, MA14, MA28,                  │  # 多尺度特征
│   year_offset,                      │  # 距2021年
│   sin(month), cos(month)            │  # 月份编码
│ ]                                   │
└─────────────────────────────────────┘
    ↓
# 趋势分支预测
MLP(20 → 64 → 32 → 26) → pred_trend_scaled
    ↓
# 反标准化
pred = scaler.inverse_transform(pred_trend_scaled)
```

## 6. 进一步融合优化建议

### 6.1 为什么简单融合不够好？

**问题分析**:
1. Model A和B相关性高 (都基于MLP)
2. Model B已经充分捕捉趋势，A提供的信息增益有限
3. 融合引入额外噪声

### 6.2 真正有效的融合方向

```
建议的异构融合架构:

┌─────────────────────────────────────────────────────────────┐
│ 基模型1: Model B (STL-Trend)                                │
│   → 负责: 趋势预测，维修感知                                │
├─────────────────────────────────────────────────────────────┤
│ 基模型2: 物理模型 (轨道劣化方程)                            │
│   → 负责: 基于材料力学的外推                                │
├─────────────────────────────────────────────────────────────┤
│ 基模型3: 统计模型 (ARIMA/指数平滑)                          │
│   → 负责: 短期波动捕捉                                      │
├─────────────────────────────────────────────────────────────┤
│ 基模型4: 大模型 (Lag-Llama/Time-MoE)                        │
│   → 负责: 复杂模式识别                                      │
└─────────────────────────────────────────────────────────────┘
                         ↓
              Stacking/Ridge融合
                         ↓
              最终预测
```

**关键**: 基模型必须异构 (不同原理)，才能互补

## 7. 实验结论

### 7.1 核心结论

1. **Model B (STL-Trend) 是当前最佳单一模型**
   - MAE = 0.2136，超越TimeMixer 27.9%
   - 关键成功因素: 显式趋势建模 + 维修感知 + 季节编码

2. **简单融合效果有限**
   - 同构模型融合收益小
   - 需要异构基模型才能互补

3. **业务特征工程至关重要**
   - 距维修年数: 最重要特征
   - 月份编码: 捕捉季节性
   - 7-9月加权: 高可信度样本
